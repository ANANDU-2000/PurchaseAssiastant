# 12 — UI Component Rules (Do/Don’t List)

## Remove (explicit)

- Excessive cards for purchase entry preview
- Duplicated totals blocks
- Giant dropdowns for units
- Manual refresh buttons
- Nested scrolling pages

## Use (required)

- Table-first purchase preview
- Inline editable cells
- Sticky bottom action bar
- Compact segmented tabs for reports
- Friendly inline errors (no raw exception dumps)

## Editing affordances

- Each row: `⋯` opens “more” sheet for charges/notes
- Tap cell to edit; numeric keyboard for numeric
