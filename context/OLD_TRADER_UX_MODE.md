# Old-trader UX mode

## Audience

Low digital literacy; prefers **large touch targets**, **high contrast**, **short text**, **no nested finance vocabulary**.

## UI requirements

- Minimum interactive height **44px** for toggles and primary buttons.
- Field borders **≥ 2px** neutral-800 on enabled, brand on focus, red on error.
- Body text **≥ 14sp** for field labels on full-page entry.
- One primary action colour (teal) — don’t mix warning colours into default fields.

## Errors

- One short sentence; map technical failures to “Check rate” / “Pick item” / “Enter qty”.

## Optional future flag

If product needs “accountant mode”, gate old GST UI behind settings — not part of default path.
